# GStack — YC Office Hours as a Claude Code Skill

## Context

From [[summary-matthew-berman-open-source-projects|Matthew Berman's open-source AI projects video]]. See also [[GStack]], [[Claude Code]].

## Content

GStack was created by Gary Tan, President of Y Combinator. It encodes his startup-building methodology as Claude Code skills. Core idea: give a solo developer the leverage of an entire development team.

**The "office hours" skill** — the most notable feature — runs 6 forcing questions that reframe your product before you write any code. Modeled on actual YC office hours sessions where partners challenge founders to stress-test their assumptions. Questions are designed to hone in on the right path forward before committing to implementation.

**Other roles included:**
- Plan / CEO review — "find the 10-star product" (Brian Chesky / Airbnb reference)
- Engineering manager, senior designer, design partner, staff engineer, debugger
- Each is a markdown prompt file that focuses Claude Code on the codebase through that role's lens

**What makes it valuable beyond the tools**: Because GStack is open source, reading the prompts is reading how Gary Tan actually thinks about startups, YC office hours, and product-market fit.

**Adoption**: ~50,000 GitHub stars within two weeks of launch.

**Installation**: Copy one prompt from docs, paste into Claude Code.
