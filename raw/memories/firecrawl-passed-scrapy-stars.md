# Firecrawl passed Scrapy in stars

## Context

From [[summary-dubibubii-claude-skills-2026|The Only Claude Skills You Need in 2026]] (2026-04-08). See [[firecrawl]].

## Content

[[firecrawl|Firecrawl]] (97K GitHub stars, YC-backed) has surpassed the star count of [Scrapy](https://scrapy.org) — the long-time default Python web scraper that has held the position for over a decade. The source's framing: *"that should tell you something."*

**Why this is an inflection point:**

- Scrapy is the legacy default for "I need to scrape data from the web with Python" — backed by years of maturity, plugin ecosystem, and community.
- Firecrawl is **purpose-built for LLM ingestion** — point at a URL, get clean markdown or structured data, no parsing required.
- Star counts are not the same as production usage, but they track **active interest**, and the crossover means more people are picking up Firecrawl-shaped tools than Scrapy-shaped ones for new work.

The wiki has tracked sister tools — [[crawl4ai]] (42K+ stars), [[stagehand]] — but Firecrawl has the largest mindshare of the agent-data category as of this source. The agent-native scraping era has overtaken the structured-data scraping era in active interest.
