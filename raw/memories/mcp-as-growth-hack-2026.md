# MCP Is the Growth Hack of 2026

## Context

From [[summary-nate-b-jones-google-stitch|Nate B Jones's Google Stitch video]]. See also [[Blender MCP]], [[Remotion]], [[Google Stitch]].

## Content

Nate B Jones's thesis from March 2026: **MCP is the growth hack of 2026**.

> "If you have a product, ask yourself: why isn't it an MCP? It should be an MCP. If it's not an MCP, you got problems."

**Why**: MCP turns any tool into an agent-accessible command-line primitive. You take a tool and say "I want to make it available at the command line" — it becomes an MCP server and agents can work with it immediately.

**Evidence from Remotion**: Remotion went from "a little website" to 150k+ Claude Code installs in 8 weeks by shipping an MCP-based skill. The product growth came from becoming agent-accessible, not from the product itself changing.

**Pattern**: MCP → skill directory (skills.sh) → viral adoption within the Claude Code user base → GitHub star growth. The path from tool to agent-accessible tool is now a standard growth playbook.

**The USB plug analogy**: MCP is the USB plug for AI. Any tool that becomes a USB-pluggable component becomes usable by every agent without custom integration.
