# Behavioral lock-in is different from data lock-in

## Context

From [[summary-nate-b-jones-conway-leak|I Analyzed 512,000 Lines of Leaked Code]]. See also [[behavioral-lock-in]], [[conway]], [[open-brain]].

## Content

**Every previous tech platform lock-in was about stuff.** Stuff is painful to migrate but technically possible — there are export tools, consultants, CSV dumps, OAuth tokens.

| Era | Lock-in surface | Migration path |
|---|---|---|
| 1990s desktop | Files (proprietary formats) | Office → LibreOffice via export |
| 2000s SaaS | Records (CRM, support, comms) | Salesforce → HubSpot via CSV |
| 2010s cloud | Workflows (integrations) | Re-architecture |
| 2026+ agents | **Behavioral context** | **None known** |

Conway-class always-on agents accumulate **the model of how you work** — which emails you triage in 5 minutes vs ignore for 3 days, which Slack threads you respond to, how your VP communicates, what a "good" PowerPoint looks like to you, when you reschedule meetings.

**The three components of a behavioral model**:
1. **Your data** — emails, calendar, Slack, docs. Portable (GDPR Art. 20, CCPA).
2. **Their compute** — the model + inference passes the agent ran. **Not portable.**
3. **Time** — months of observing what you actually do with surfaced items. **Not portable.**

Nate B Jones's frame: *"There's no CSV of how this person thinks. There's no migration consultant for behavioral context. When you switch away from Conway after 6 months, you don't just lose an agent — you lose the 6 months of compounding that made the agent useful. You're back to a brilliant stranger you have to explain everything to."*

**The structural counter** (developed in [[open-brain]]): user-owned context infrastructure exposed through MCP. The memory lives in a database the user controls; switching providers loses runtime intelligence but preserves accumulated context. Convenience will still win for most users — but the structural distinction matters at the edge (regulated industries, employees changing jobs).

**The unresolved policy question**: how should the value generated by behavioral evidence be allocated back to the worker? Most companies will default to "whatever you do at work belongs to the company." Nate's argument: portability frameworks should ship before Conway does, not after.
