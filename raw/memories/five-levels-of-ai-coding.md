# Five Levels of AI Coding — Dan Shapiro's Framework

## Context

From [[summary-nate-b-jones-five-levels|Nate B Jones's 5 Levels video]]. See also [[Five Levels of AI Coding]], [[Attractor]].

## Content

Dan Shapiro's framework: L0 spicy autocomplete, L1 coding intern, L2 junior dev (90% of "AI-native" devs stuck here), L3 dev as manager, L4 dev as PM, L5 dark factory (spec in → working software out, lights off).

**StrongDM at L5**: 3 engineers, Attractor agent (3 markdown spec files = the entire agent), external scenarios (holdout set — agent never sees eval criteria), digital twin universe (simulated Okta/Jira/Slack/Google). Output: 16K lines Rust, 9.5K Go, 700 TypeScript — real production software built entirely by agents.

**The J-curve**: METR RCT found experienced devs 19% SLOWER with AI. They believed 24% faster. Most orgs at the bottom of the J, interpreting the dip as evidence AI doesn't work. The fix isn't better tools — it's end-to-end process transformation.

**Bottleneck shift**: Implementation speed → spec quality. "Adequate is no longer a viable career position because adequate is what the models do."
